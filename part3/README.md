Part 3 - Advanced Modeling - Ensembles, Tuning, and Full ML Pipeline

A baseline Decision tree classifier is trained using the default settings max_depth=None. The model achieved 100% training accuracy and 85.07% testing accuracy. The perfect training accuracy with the lower testing accuracy shows that the model is overfitting. This means it has learned the training data too closely and does not perform as well on new or unseen data. Decision trees are high-variance models because they choose the best split at each step based only on the training data. Since earlier decisions cannot be changed, even small changes in the training data can create a very different tree, making the model more likely to overfit.

The second decision tree classifier is trained using max_depth=5 and min_samples_split=20 to control the complexity of the model. The controlled tree achieved 92.99% training accuracy and 93.28% testing accuracy. Compared with the baseline decision tree performance of 100% training and 85.07% testing accuracy, the controlled model shows much better generalization and does not exhibit noticeable overfitting. The max_depth parameter limits how deep the tree can grow and reducing the chance of learning unnecessary patterns from the training data. The min_samples_split parameter requires at least 20 samples before a node can be split - preventing the tree from creating splits based on very small groups of data that may represent noise. As a result the controlled tree produces a much smaller train-test accuracy gap and performs more consistently on unseen data.

Two decision tree models were trained using Gini impurity and Entropy as the splitting criteria while keeping max_depth=5 the same. Both models achieved the same test accuracy of 92.16% indicating that the choice of splitting criterion had no noticeable effect on the model's performance for this dataset. 

Gini = 1 − Σ(pi²)
where *pi* is the probability of class *i*

Entropy = − Σ(pi × log₂(pi))
where *pi* is the probability of class *i*

Gini impurity of 0 means that all the samples in a node belong to the same class, so the node is completely pure. Both Gini and Entropy try to find the best split by creating purer child nodes but they calculate impurity in different ways. In this dataset both methods achieved the same test accuracy of 92.16% which shows that either criterion can be used without changing the model's performance.

Random forest classifier is trained using 100 decision trees with a maximum depth of 10. The model achieved a training accuracy of 96.17%, testing accuracy of 93.66% and ROC-AUC of 0.9555. The small difference between the training and testing accuracy indicates that the model generalizes well to unseen data and does not show significant overfitting.

Random forest model showed that age (0.4409) is the most important feature for predicting insurance charges followed by smoking status (0.2990). Then the followed important features are BMI (0.0593), annual income (0.0575), and blood pressure (0.0544). This means that a person's age and smoking habit have the biggest impact on whether their insurance charges are high or low. BMI, annual income, and blood pressure also help in making predictions but their influence is less than age and smoking status.

Random Forest measures feature importance by checking how much each feature helps reduce Gini impurity when splitting the data across all the trees in the forest. Features that reduce impurity more often receive higher importance scores. Unlike Linear Regression coefficients - feature importance does not show whether a feature increases or decreases the prediction. It only shows how useful that feature is for making accurate predictions.

Random Forest uses a method called bagging - Bootstrap Aggregating. Instead of training all trees on the same data, each decision tree is trained using a different random sample of the training data selected with replacement. At each split, the model also considers only a random subset of features (about 4 out of 13 features in this dataset). Because each tree learns from different data and different features the trees make different prediction errors. The final prediction is made by combining the predictions from all the trees. This reduces overfitting, improves stability, and usually gives better performance than a single decision tree.

A Gradient boosting classifier was trained using 100 estimators, learning rate of 0.1, and maximum tree depth of 3. The model achieved a training accuracy of 95.89%, testing accuracy of 93.66%, and ROC-AUC of 0.9633. The small difference between the training and testing accuracy indicates that the model generalizes well and does not show significant overfitting. Among the models evaluated, Gradient boosting achieved the highest ROC-AUC - indicating excellent ability to distinguish between high and low insurance charges.

Based on the Random forest feature importance scores the five least important features are region_northwest, region_southeast, region_southwest, pre_existing_condition, and sex_male. These features are removed, and a second random forest model was trained using the same hyperparameters. The full model achieved a ROC-AUC of 0.9555, while the reduced model achieved a slightly higher ROC-AUC of 0.9600. This indicates that the removed features were largely uninformative and may have introduced unnecessary complexity or noise into the model. Removing these features simplified the model without reducing its predictive performance. In a production environment, using a simpler model can reduce computation time, storage requirements, and maintenance effort. Since the model's performance is maintained and even improved slightly, removing these low-importance features would be a reasonable choice for deployment.

A 5-fold Stratified cross-validation is carried out to compare the four models. Among them Gradient boosting gave the best performance with a mean ROC-AUC of 0.9501 and a standard deviation of 0.0166. This means it was the most accurate model and also produced consistent results across the five folds. Random forest also performed well with a mean ROC-AUC of 0.9479 and a standard deviation of 0.0170 followed by logistic regression, which achieved a mean ROC-AUC of 0.9466 with the same standard deviation of 0.0170. The Decision Tree achieved a mean ROC-AUC of 0.9409 and a standard deviation of 0.0168, making it the lowest-performing model among the four - although its performance is still good. 

Cross-validation gives a more reliable measure of model performance because every data sample is used for both training and testing in different folds. This reduces the effect of depending on a single train-test split and provides a better estimate of how the models are likely to perform on new, unseen data. The standard deviation shows how consistent the model's performance is across the five folds. Since all the standard deviation values are low of about 0.017, the models showed stable and consistent performance.

Grid search iss performed to find the best hyperparameters for the Random forest model. A pipeline containing a SimpleImputer, StandardScaler, and RandomForestClassifier is used so that preprocessing is applied consistently during every cross-validation fold. The search evaluated 18 different hyperparameter combinations, resulting in 90 model fits using 5-fold Stratified Cross-Validation. The best hyperparameter combination:Number of trees (n_estimators):200; Maximum depth (max_depth):none; Minimum samples per leaf (min_samples_leaf):1. Using these parameters the model achieved the highest mean cross-validation ROC-AUC of 0.9516. Grid search tests every possible combination of the specified hyperparameters, making it more likely to find the best-performing model. However it can take a long time when many parameters are included. Randomized search evaluates only a random selection of parameter combinations. This makes it much faster for large search spaces, but it may not always find the optimal combination.

Manual learning curve is created by training the best random forest pipeline on 20%, 40%, 60%, 80%, and 100% of the training data. For each training size the training ROC-AUC and test ROC-AUC are calculated - The training AUC remained at 1.0 for all training sizes, showing that the Random Forest model was able to fit the training data almost perfectly even as more data was added. Therefore the training AUC did not decrease as the training set grew. The test AUC remained relatively stable, ranging from 0.9524 to 0.9596, with only small fluctuations as the amount of training data increased. There was no consistent upward trend, suggesting that adding more training data did not significantly improve the model's performance. Based on these results, the model appears to be limited more by model capacity than by data quantity. Since the test AUC has largely plateaued at the full training size, collecting additional data alone is unlikely to produce substantial improvement. Further gains would more likely come from trying different models, feature engineering, or additional hyperparameter tuning.

The best Random Forest pipeline obtained from GridSearchCV was saved as best_model.pkl using joblib.dump(). The saved model is successfully reloaded using joblib.load(), and predictions are generated for two manually created test samples. The model predicted the classes [0, 1] without any errors confirming that the serialized model can be reused for future predictions without retraining.
