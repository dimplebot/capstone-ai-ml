The feature matrix x is created by selecting all columns expect charges. Hence resulting with 11 input feature attributes and 1338 records. The regression target y_reg is defined as the continuous charges column, which represents the individual medical insurance cost. For the classification task, a binary target y_clf is created by comparing each insurance charge with the dataset's median value (9382.033). Records with charges less than or equal to the median were assigned as 0 (Low Charge), while those greater than the median were assigned as 1 (High Charge). This resulted in a perfectly balanced dataset with 669 samples in each class, making it suitable for training and evaluating classification models.

Categorical features are encoded based on whether they had a natural order. The exercise_frequency column is label encoded using the order Never < Rarely < Weekly < Daily, and occupation_risk is encoded as Low < Moderate < High because these categories represent increasing levels of frequency and risk. The remaining categorical columns sex, smoker, and region do not have a natural order, so they were converted using one-hot encoding with the first category dropped. This prevents the model from assuming a false numerical relationship between categories and also avoids multicollinearity by removing one redundant dummy variable.
If label encoding had been used for these columns, the model might incorrectly assume that one category is greater or smaller than another (for example, Northwest > Southeast), even though no such relationship exists. One-hot encoding represents each category as a separate binary feature, avoiding this false ordinal relationship. The first dummy column was dropped to prevent multicollinearity.

The dataset was split into 80% training of about 1,070 records and 20% testing of about 268 records using train_test_split() with random_state=42 to ensure reproducible results. The features were standardized using StandardScaler, which was fitted only on the training data and then applied to both the training and testing sets. Fitting the scaler on the entire dataset would cause data leakage, because the mean and standard deviation of the test set would be used during training. This would allow the model to indirectly learn information from the test data, leading to overly optimistic and unreliable performance.

The linear regression model was trained using the standardized training data to predict insurance charges. The model achieved a Mean Squared Error (MSE) of 33,999,438.03 and an R² score of 0.7810, which means it explains about 78.1% of the variation in insurance charges. This indicates that the model predicts insurance charges reasonably well.

The three features with the highest impact on the prediction were smoker_yes (9608.72), age (3611.37), and bmi (2067.22). This shows that smoking status has the greatest influence on insurance charges, followed by age and body mass index (BMI).

A positive coefficient means that as the value of a feature increases then the predicted insurance charge also increases. For eg - the large positive coefficient for smoker_yes indicates that smokers are expected to have much higher insurance charges than non-smokers. A negative coefficient means that increasing the value of that feature is associated with a decrease in the predicted insurance charges.

A ridge regression model with alpha 1.0 was also trained using the same training and testing data. It achieved an MSE of 34,004,839.29 and an R² score of 0.7810, which is almost the same as the Linear Regression model.

Ridge Regression uses L2 regularization to reduce the size of the model coefficients helping to make the model more stable and reduce overfitting. The alpha parameter controls how much regularization is applied. A higher alpha applies stronger regularization and a lower alpha makes the model behave more like standard Linear Regression. Since both models produced very similar results, regularization had only a small effect on this dataset.

precision = TP/(TP+FP)
recall = TP/(TP+FN)

For this dataset and problem type - Recall is more important than Precision. A false negative means that a person with high insurance charges is incorrectly classified as having low charges. Missing these high-cost cases could lead to underestimating healthcare costs. so it is more important to identify as many high-charge individuals as possible.

The ROC curve shows how well the model distinguishes between the two classes at different decision thresholds. The model achieved an AUC of about 0.9603 which indicates good classification performance. An AUC close to 1.0 means the model can accurately separate low-charge and high-charge individuals, while AUC of 0.5 implies that the classifier lacks the ability to distinguish between positive and negative classes effectively. The ROC curve lies close to the top-left corner, showing a high true positive rate with a low false positive rate across different thresholds.

The Logistic Regression model is evaluated using decision thresholds from 0.30 to 0.70. As the threshold increased, precision improved, while recall gradually decreased. This shows the trade-off between correctly identifying high-charge individuals and reducing false positive predictions. The highest F1-score of about 0.9378 is obtained at a threshold of 0.60 indicating the best balance between precision and recall for this dataset.

For this classification task, Recall is considered more important because missing a person with high insurance charges (false negative) could underestimate future healthcare costs. However the 0.60 threshold improved precision while maintaining the same recall as the default 0.50 threshold, it provides a better overall balance for this dataset. Increasing the threshold further to 0.70 improves precision but reduces recall, meaning more high-charge individuals could be missed.

Recall is more important for this classification task because missing a person with high insurance charges (false negative) could underestimate future healthcare costs. A lower threshold of 0.30 achieves the highest recall (0.9426), meaning more high-charge individuals are correctly identified. However, this also reduces precision (0.7667) and F1-score (0.8456) because more low-charge individuals are incorrectly classified as high-charge. Although lowering the threshold improves recall, a threshold of 0.60 provides the best overall balance, achieving the highest F1-score (0.9378) while maintaining the same recall as the default threshold 0.50 and improving precision. So if maximizing recall alone is the priority then 0.30 would be preferred, but for overall model performance 0.60 is the better choice.

A second Logistic Regression model was trained with C=0.01 to apply stronger L2 regularization. The C parameter controls the amount of regularization applied to the model. A smaller value of C applies stronger regularization, reducing the size of the model coefficients and helping to prevent overfitting. A larger value of C applies less regularization, allowing the model to fit the training data more closely. The performance of the two models was compared using precision, recall, and AUC. The baseline model with C=1.0 achieved higher precision 0.9040 and a slightly higher AUC 0.9603 than the model with C=0.01, while recall remained the same 0.9262 for both models. This indicates that reducing C did not improve performance on this dataset. Therefore, the baseline Logistic Regression model with C=1.0 was selected as the better model.

A bootstrap analysis with 500 samples drawn with replacement was performed using np.random.choice() to compare the AUC values of the Logistic Regression models with C=1.0 and C=0.01. The mean AUC difference (C=1.0 − C=0.01) was 0.001653. The 95% confidence interval, calculated using the 2.5th and 97.5th percentiles, ranged from -0.001234 to 0.004809. Since the confidence interval includes zero, the performance difference between the two models is not statistically reliable. This indicates that the small improvement observed with the C=1.0 model may be due to random sampling variation rather than a consistent advantage across different test samples.
